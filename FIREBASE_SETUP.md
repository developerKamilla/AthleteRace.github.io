# Athlete Race — подключение реального backend-чата

В проект уже встроена поддержка Firebase Realtime Database. После настройки чат, дневник тренера, веса, пользователи, записи на тренировки и события будут синхронизироваться между разными телефонами и ноутбуками.

## 1. Создайте проект Firebase

1. Откройте Firebase Console.
2. Нажмите **Add project**.
3. Создайте проект, например `athlete-race`.
4. В Project settings добавьте Web App.
5. Скопируйте объект `firebaseConfig`.

## 2. Вставьте config

Откройте файл:

```txt
firebase-config.js
```

И замените значения `PASTE_...` на реальные значения из Firebase.

## 3. Включите Realtime Database

1. Firebase Console → **Build** → **Realtime Database**.
2. Нажмите **Create Database**.
3. Выберите регион, лучше ближайший к пользователям.
4. Для быстрого теста можно включить test mode.

## 4. Rules для теста

Для тестового режима можно вставить правила из `database.rules.json`.

Важно: эти правила открытые для демо. Для продакшена лучше подключить Firebase Auth и ограничить запись только авторизованным пользователям.

## 5. Проверка

1. Запустите проект на одном устройстве.
2. Войдите одним пользователем.
3. Откройте тот же проект на другом устройстве или в другом браузере.
4. Войдите другим пользователем.
5. Напишите сообщение в чат — оно должно появиться у обоих.

В чате должен появиться статус **Realtime sync online**. Если написано **Local demo mode**, значит config не вставлен или Realtime Database не включена.
